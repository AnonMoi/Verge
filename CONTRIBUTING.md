# Verge 贡献指南

感谢你关注 **Verge**。Verge 是一款基于 Godot 4.x 开发的 2D 时间循环塔防游戏，目前仍处于积极开发阶段。

本指南面向希望参与玩法、关卡、程序、美术、音频、叙事、测试或文档工作的贡献者，旨在保持项目质量、开发节奏与游戏体验的一致性。

> 当前 `master` 是个人仓库的主开发分支。请不要直接向 `master` 推送改动；所有贡献应通过功能分支和 Pull Request 提交。

---

## 贡献方向

你可以参与以下工作：

- **玩法与系统**：时间循环、战斗、阻挡、经济、角色与敌人系统
- **关卡与内容**：关卡配置、敌人波次、数值平衡、剧情与引导
- **程序与工具**：GDScript、Godot 场景、UI、性能优化、开发工具
- **美术与音频**：角色、敌人、场景、特效、界面、音乐与音效
- **质量与文档**：Bug 复现、兼容性测试、开发文档与使用说明

## 行为准则

- 尊重所有贡献者，保持专业、友好的沟通
- 讨论设计和代码，不针对个人发表评价
- 接受建设性反馈，以玩家体验和项目目标为优先
- 提交自己拥有合法使用权的代码、素材和音频
- 未经授权，不提交包含个人隐私、密钥或其他敏感信息的内容

## 如何贡献

### 1. 报告 Bug

如果你发现了 Bug，请在 GitHub Issues 中提交，并包含：

- Bug 的简要描述
- 复现步骤（越详细越好）
- 预期行为和实际行为
- 截图（如有）
- 你的操作系统和 Godot 版本

### 2. 提出新功能

新功能建议同样通过 Issue 提交，说明：

- 这个功能解决什么问题
- 建议的实现方式（可选）
- 对现有功能的影响

### 3. 提交代码

#### 开发流程

```bash
# 1. 克隆个人仓库
git clone https://github.com/AnonMoi/Verge.git
cd Verge

# 2. 基于最新 master 创建工作分支
git switch master
git pull origin master
git switch -c feature/你的功能名

# 3. 在 Godot 中开发并进行本地验证

# 4. 提交改动
git add -A
git commit -m "feat: 简要描述你的改动"

# 5. 推送工作分支
git push -u origin feature/你的功能名

# 6. 在 GitHub 上创建 Pull Request，目标分支选择 master
```

#### 分支命名

分支名称应简短、明确，并使用以下前缀：

- `feature/功能描述` — 新玩法或新功能
- `fix/问题描述` — Bug 修复
- `balance/数值描述` — 数值与平衡调整
- `art/内容描述` — 美术、特效或音频资源
- `docs/文档描述` — 文档修改
- `chore/事项描述` — 工具、CI 或维护性改动

#### 提交信息规范

推荐使用 Conventional Commits 风格，并在必要时补充范围：

- `feat: 添加章节选择剧情入口`
- `fix: 修复胜利结算面板按钮无法点击`
- `balance: 调整杂兵生命值与击杀奖励`
- `art: 更新胜利音效`
- `docs: 更新贡献指南`
- `ci: 调整 Godot 自动导出工作流`

#### Pull Request 要求

每个 PR 应当：

- 使用清晰的标题，说明改动类型和内容
- 描述改了什么、为什么改，以及如何验证
- 关联对应的 Issue（如 `Closes #12` 或 `Refs #12`）
- 保持范围小且专注，避免把无关改动混在一起
- 确保代码在 Godot 4.x 中可以正常运行
- 对玩法、数值或资源改动说明对玩家体验的影响
- AI 辅助生成的代码必须经过实际运行和人工检查
- 不提交密钥、个人信息、未经授权的资源或大体积临时文件

#### 合并前检查

提交 PR 前请确认：

- [ ] 本地可以打开项目并运行主要流程
- [ ] 相关场景、脚本和资源路径正常
- [ ] 已验证新增或修改的玩法
- [ ] 已检查调试代码、临时文件和无关改动
- [ ] GitHub Actions 检查通过（如适用）
- [ ] PR 描述和截图能够帮助 Reviewer 理解改动

### 4. 代码审查

- PR 默认合入 `master`，未经 Review 不直接合并
- Review 应关注正确性、可维护性、玩家体验和潜在回归
- Review 时请提出具体的问题或建议，不要只说“LGTM”
- 根据 Review 意见修改后，继续推送到同一分支，PR 会自动更新

## 开发环境

- **引擎**：Godot 4.x（推荐 4.7）
- **语言**：GDScript
- **渲染器**：gl_compatibility
- **分辨率**：1280×720

## 维护者决策

项目维护者会根据以下因素评估贡献：

- 是否符合 Verge 的玩法方向和视觉风格
- 是否改善稳定性、可维护性或玩家体验
- 是否具备清晰的测试信息和变更范围
- 是否引入新的版权、隐私或技术风险

维护者可以要求修改、暂缓或拒绝不符合项目目标的贡献。感谢你的理解与参与。
